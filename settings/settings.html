    Custom Filter Rules

    One rule per line. Use AdBlock-style syntax: ||example.com^ to block, @@||example.com^ to allow.

    Rule Format Guide

    ||example.com^Block all requests to example.com
    @@||example.com^Allow (whitelist) example.com
    ||example.com/ads/Block a specific path
    /banner_ad/Block URLs containing "banner_ad"
    ! commentComment line (ignored)

    About ShieldBlock

    ShieldBlock

    Version 1.0.0

    A production-grade, privacy-first ad blocker using Chrome's declarativeNetRequest API. Fully self-contained — no external dependencies or servers.

    Features

    • 220+ built-in filter rules (ads, trackers, miners)
    • Cosmetic filtering — hides ad elements that slip through
    • Cookie consent banner auto-removal (50+ frameworks)
    • Anti-adblock bypass — bait elements + overlay removal
    • Per-site whitelist with one click
    • Custom filter rules (AdBlock syntax)
    • Turkish + English UI support
    • Dark / light mode

    Permissions Used

    • declarativeNetRequest — Block network requests via rule engine
    • storage — Save settings and statistics locally
    • tabs / activeTab — Read current tab URL for per-site toggle
    • scripting — Inject cosmetic filter content scripts
    • webNavigation — Detect page navigation for badge reset
